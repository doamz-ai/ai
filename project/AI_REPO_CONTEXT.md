# AI Repo Context

本文件记录当前代码仓库的项目画像。

它不是通用系统规则，而是每个具体项目安装 AI Engineering Delivery OS 后必须生成和维护的“项目事实地图”。

AI 在理解、规划、执行、审计任何任务前，都应优先阅读本文件。

---

## 0. Maintenance Rules

### 0.1 Source of truth

本文件必须基于真实仓库事实维护，包括：

- README / docs。
- 代码目录结构。
- 配置文件。
- 数据库 schema / migration。
- API 路由。
- 前端页面。
- 测试和部署配置。
- 已合并的历史任务记录。

禁止把未确认的推测写成事实。

### 0.2 Evidence labels

建议使用以下标签：

```text
[Confirmed] 已从代码或文档确认。
[Assumption] 当前基于合理假设。
[To Verify] 需要后续智能体进一步确认。
[Unknown] 当前未知。
```

### 0.3 Update timing

以下情况应更新本文件：

- 项目技术栈变化。
- 核心目录结构变化。
- 核心业务流程变化。
- API / 数据库 / 权限模型发生重要变化。
- 文件存储、部署、任务队列等基础设施发生变化。
- 高风险区域发生变化。
