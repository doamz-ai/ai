# Security Checklist

本清单用于所有涉及权限、用户数据、文件、API、外部服务、密钥、下载、上传、后台管理的任务。

目标：防止 AI 编程过程中遗漏后端鉴权、资源归属、输入校验、敏感信息保护和文件访问安全。

---

## 1. When to Use

涉及以下任一情况时必须使用：

- 登录 / 注册 / session / token。
- 用户角色、管理员、子用户、租户隔离。
- 资源详情、列表、下载、导出、删除、更新。
- 文件上传 / 下载 / 对象存储。
- 外部 API key / secret / webhook。
- 后台管理页面。
- 数据库查询、动态 SQL、搜索条件。
- 命令执行、路径拼接、临时文件。

---

## 2. Permission and Authorization

- [ ] 后端是否做了权限校验，而不是只靠前端隐藏？
- [ ] 是否检查当前用户是否有权访问该资源？
- [ ] 管理员能力是否显式校验？
- [ ] 普通用户是否无法访问他人数据？
- [ ] 列表接口是否只返回当前用户可见数据？
- [ ] 详情接口是否检查资源归属？
- [ ] 下载 / 导出接口是否检查资源归属？
- [ ] 更新 / 删除接口是否检查资源归属？
- [ ] 是否避免通过猜 ID、路径、object key 访问他人资源？

---

## 3. Authentication

- [ ] 受保护接口是否要求登录？
- [ ] session / token 是否从可信来源读取？
- [ ] token 过期、无效、缺失时是否有明确错误？
- [ ] 是否避免把认证信息暴露给前端不该看到的地方？
- [ ] 登出或失效逻辑是否一致？

---

## 4. Input Validation

- [ ] 所有外部输入是否经过校验？
- [ ] 数字、日期、枚举、ID、分页参数是否校验范围？
- [ ] 字符串是否限制长度？
- [ ] 文件名、路径、URL 是否防注入？
- [ ] 是否避免直接拼接 SQL？
- [ ] 是否避免直接拼接 shell 命令？
- [ ] 是否避免把用户输入直接用于文件路径？

---

## 5. Data Exposure

- [ ] API 返回是否只包含必要字段？
- [ ] 是否避免返回密码、token、secret、内部配置？
- [ ] 错误信息是否避免泄露内部路径、SQL、密钥、堆栈？
- [ ] 日志是否避免输出敏感数据？
- [ ] 前端是否避免把敏感数据存入可公开访问位置？

---

## 6. File and Object Storage Security

- [ ] 上传文件是否限制类型、大小、扩展名？
- [ ] 文件名是否规范化？
- [ ] 临时文件是否有清理策略？
- [ ] 下载是否经过后端鉴权？
- [ ] 对象存储链接是否避免长期公开？
- [ ] 数据库是否优先保存 object key 而不是永久公开 URL？
- [ ] 是否防止路径穿越？
- [ ] 是否防止用户下载他人文件？

---

## 7. Secrets and Configuration

- [ ] secret 是否通过环境变量或安全配置管理？
- [ ] secret 是否没有写入代码？
- [ ] secret 是否没有写入日志？
- [ ] secret 是否没有打包进前端？
- [ ] 新增环境变量是否有文档说明？
- [ ] 是否避免把真实密钥提交到仓库？

---

## 8. Database Security

- [ ] 查询是否参数化？
- [ ] 是否避免 SQL 注入？
- [ ] 是否限制列表查询范围和分页？
- [ ] 是否避免越权聚合统计？
- [ ] 是否避免普通用户通过筛选条件访问他人数据？

---

## 9. External Integrations

- [ ] 外部 API 调用是否有超时？
- [ ] 外部 API 失败是否安全处理？
- [ ] webhook 是否校验签名？
- [ ] 回调接口是否避免未授权访问？
- [ ] 外部错误是否不会泄露内部敏感信息？

---

## 10. Review Gate Questions

审计时必须回答：

1. 是否存在越权访问风险？
2. 是否存在敏感数据泄露风险？
3. 是否存在文件泄露或路径穿越风险？
4. 是否存在注入风险？
5. 是否存在密钥泄露风险？
6. 是否只做了前端权限隐藏而没有后端校验？

---

## 11. If Risk Is Found

如果发现安全风险，应输出：

```text
Severity: Critical / Important / Optional
Risk:
Affected area:
Required fix:
Acceptance criteria:
```

Critical 安全问题不得交付。
