# Frontend UX Checklist

本清单用于前端页面、组件、表单、列表、导航、权限显示和用户反馈相关任务。

目标：防止 AI 只实现 happy path，遗漏 loading、empty、error、permission denied、响应式和交互一致性。

---

## 1. When to Use

涉及以下情况时必须使用：

- 新页面。
- 新组件。
- 表单。
- 列表 / 表格。
- 筛选 / 搜索 / 分页。
- 权限差异展示。
- 下载 / 导出按钮。
- 错误提示。
- 菜单 / 路由。

---

## 2. Consistency

- [ ] 是否复用现有布局？
- [ ] 是否复用现有组件？
- [ ] 是否符合现有视觉风格？
- [ ] 是否符合现有路由和菜单风格？
- [ ] 是否避免引入新 UI 框架？
- [ ] 是否避免重复造组件？

---

## 3. Page States

必须考虑：

- [ ] Loading。
- [ ] Empty。
- [ ] Success。
- [ ] Error。
- [ ] Permission denied。
- [ ] Partial result。
- [ ] Disabled / unavailable。

---

## 4. Forms and Inputs

- [ ] 必填项是否校验？
- [ ] 输入格式是否校验？
- [ ] 错误提示是否清晰？
- [ ] 提交中是否防重复提交？
- [ ] 提交失败是否保留用户输入？
- [ ] 成功后是否有反馈？

---

## 5. Lists and Tables

- [ ] 是否分页？
- [ ] 是否支持必要筛选？
- [ ] 是否有空状态？
- [ ] 是否有加载状态？
- [ ] 是否有错误状态？
- [ ] 大数据量是否有性能风险？
- [ ] 行操作是否有权限控制？

---

## 6. Permission UX

- [ ] 不同角色看到的入口是否合理？
- [ ] 无权限时是否有明确提示？
- [ ] 前端隐藏是否不替代后端权限？
- [ ] 管理员和普通用户视角是否分清？

---

## 7. API Interaction

- [ ] API 调用是否复用现有封装？
- [ ] 错误是否统一处理？
- [ ] loading 状态是否正确？
- [ ] 重试或刷新是否合理？
- [ ] 下载 / 导出是否处理失败？

---

## 8. Accessibility and Usability

- [ ] 按钮文案是否清晰？
- [ ] 危险操作是否二次确认？
- [ ] 表格、表单是否可读？
- [ ] 重要操作是否有反馈？
- [ ] 是否避免使用只有颜色才能区分的状态？

---

## 9. Responsive / Layout

- [ ] 常见屏幕尺寸是否可用？
- [ ] 表格是否溢出处理？
- [ ] 关键按钮是否不会被遮挡？
- [ ] 长文本是否处理换行或省略？

---

## 10. Testing / Verification

- [ ] 正常路径。
- [ ] 空数据。
- [ ] API 失败。
- [ ] 无权限。
- [ ] 表单校验。
- [ ] 分页 / 筛选。
- [ ] 下载 / 导出失败。

---

## 11. Review Gate Questions

1. 页面是否符合项目现有风格？
2. 是否处理 loading / empty / error？
3. 是否处理权限差异？
4. 是否避免只靠前端做权限？
5. 大列表是否有性能风险？
6. 用户失败时是否知道发生了什么？
