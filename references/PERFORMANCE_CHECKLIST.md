# Performance Checklist

本清单用于涉及查询、大数据、批处理、文件生成、API 响应、外部服务调用、前端列表渲染和异步任务的改动。

目标：防止 AI 在功能实现中引入慢查询、内存暴涨、超时、大响应、无分页和不可控并发。

---

## 1. When to Use

涉及以下情况时必须使用：

- 列表查询。
- 搜索 / 筛选 / 排序。
- 数据库聚合。
- 大文件导入 / 导出。
- 批量任务。
- 外部 API 调用。
- 前端大表格或大量数据渲染。
- 对象存储上传 / 下载。
- 后台任务或定时任务。

---

## 2. Database / Query Performance

- [ ] 列表接口是否分页？
- [ ] 是否避免一次性查询全部数据？
- [ ] 是否避免 N+1 查询？
- [ ] 是否对常用筛选字段建立索引？
- [ ] 排序字段是否可支持索引？
- [ ] 是否避免在大表上做无索引模糊查询？
- [ ] 是否避免把复杂计算放在高频请求路径？
- [ ] 是否限制用户可请求的数据范围？
- [ ] 是否有合理默认 page_size 和最大 page_size？

---

## 3. API Response Performance

- [ ] API 是否只返回必要字段？
- [ ] 大结果是否通过文件、分页或异步任务返回？
- [ ] 是否避免把大数组直接塞进响应？
- [ ] 是否避免返回大型嵌套对象？
- [ ] 是否有响应大小风险？
- [ ] 是否有超时风险？

---

## 4. File / Export Performance

- [ ] 大文件是否流式处理或分批处理？
- [ ] 是否避免一次性把大文件读入内存？
- [ ] 文件生成是否有临时目录和清理策略？
- [ ] 导出任务是否可能需要异步化？
- [ ] 对象存储上传失败是否可追踪？
- [ ] 是否避免在同步请求中执行长时间导出？

---

## 5. External Service Calls

- [ ] 外部调用是否有超时？
- [ ] 是否有重试策略或失败记录？
- [ ] 是否限制并发？
- [ ] 是否避免在循环中串行调用大量外部 API？
- [ ] 是否考虑限流和成本？
- [ ] 外部服务失败是否不会阻塞整个系统？

---

## 6. Frontend Performance

- [ ] 大列表是否分页、虚拟滚动或按需加载？
- [ ] 是否避免一次性渲染大量 DOM？
- [ ] 是否避免频繁重复请求？
- [ ] 是否有 loading / empty / error 状态？
- [ ] 是否避免把大文件内容塞进前端内存？
- [ ] 是否避免无防抖的搜索请求？

---

## 7. Background Jobs

- [ ] 长任务是否应放入后台任务？
- [ ] 任务状态是否可追踪？
- [ ] 失败是否可记录？
- [ ] 是否避免重复提交？
- [ ] 是否有任务清理策略？
- [ ] 是否有并发控制？

---

## 8. Caching

- [ ] 是否存在可缓存的重复计算？
- [ ] 缓存是否有过期策略？
- [ ] 缓存是否会造成权限数据泄露？
- [ ] 缓存 key 是否包含必要的用户/租户/权限维度？
- [ ] 是否避免过早引入复杂缓存？

---

## 9. Review Gate Questions

审计时必须回答：

1. 是否会在大数据量下明显变慢？
2. 是否存在无分页大查询？
3. 是否存在内存暴涨风险？
4. 是否存在同步长任务阻塞请求？
5. 是否存在外部服务调用失控？
6. 是否引入了无必要的复杂缓存或异步系统？
