# Database Migration Checklist

本清单用于所有涉及数据库 schema、表、字段、索引、状态、迁移脚本和数据兼容性的任务。

目标：防止 AI 直接改数据结构而没有 migration、兼容性、回滚和历史数据考虑。

---

## 1. When to Use

涉及以下情况时必须使用：

- 新增表。
- 新增字段。
- 修改字段类型或语义。
- 删除字段或表。
- 新增索引。
- 新增状态字段。
- 数据迁移。
- ORM model 变更。
- 查询逻辑依赖新字段。

---

## 2. Schema Change

- [ ] 是否需要 migration？
- [ ] 是否使用项目现有 migration 工具？
- [ ] model 和 schema 是否同步？
- [ ] 字段类型是否合适？
- [ ] 字段默认值是否明确？
- [ ] 字段是否允许 null？
- [ ] 是否需要唯一约束？
- [ ] 是否需要外键或逻辑关联？
- [ ] 是否需要索引？

---

## 3. Backward Compatibility

- [ ] 是否影响旧代码读取？
- [ ] 是否影响旧数据？
- [ ] 新字段为空时系统是否正常？
- [ ] 是否需要 backfill？
- [ ] 是否能灰度上线？
- [ ] 回滚后是否会有数据不兼容？

---

## 4. Data Migration

- [ ] 是否需要迁移历史数据？
- [ ] 数据量是否可控？
- [ ] 迁移是否可重复执行？
- [ ] 迁移失败是否可恢复？
- [ ] 是否需要分批迁移？
- [ ] 是否需要备份？

---

## 5. Index and Query Impact

- [ ] 新查询是否需要索引？
- [ ] 索引是否会影响写入性能？
- [ ] 是否存在慢查询风险？
- [ ] 是否涉及大表？
- [ ] 是否需要分页或限制？

---

## 6. State Fields

如果新增状态字段：

- [ ] 状态枚举是否明确？
- [ ] 初始状态是什么？
- [ ] 状态流转是否定义？
- [ ] 是否有非法状态保护？
- [ ] 失败状态是否记录原因？
- [ ] 是否需要 finished_at / updated_at 等时间字段？

---

## 7. Rollback

- [ ] 是否有 rollback migration？
- [ ] 回滚是否会丢数据？
- [ ] 如果不能安全回滚，是否明确说明？
- [ ] 是否需要人工备份？

---

## 8. Testing

- [ ] migration 能否执行成功？
- [ ] 新 schema 下旧功能是否正常？
- [ ] 新功能是否能读写数据？
- [ ] 空值、默认值、历史数据是否验证？
- [ ] 回滚是否验证？

---

## 9. Documentation

- [ ] 是否更新数据模型说明？
- [ ] 是否更新 API / service 说明？
- [ ] 是否更新环境或部署步骤？
- [ ] 是否需要记录到 AI_PROJECT_MEMORY？
- [ ] 是否需要 ADR？

---

## 10. Review Gate Questions

1. 是否有 migration？
2. 是否考虑历史数据？
3. 是否考虑回滚？
4. 是否考虑索引和查询性能？
5. 是否会破坏旧功能？
6. 是否测试了 migration？
